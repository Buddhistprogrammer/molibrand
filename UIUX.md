## 1. 设计理念

### 1.1 核心理念：「专业可信，温暖引导」

本产品面向中小企业老板和创业者——他们是非技术人群，对品牌咨询有需求但缺乏专业知识，决策门槛高（品牌咨询是高信任决策）。因此设计的核心目标是：

> **让用户在第一眼就感到"这个工具懂我，且值得信任"，在使用中感到"原来品牌咨询可以这么简单"。**

### 1.2 设计原则

| 原则 | 说明 | 设计映射 |
|------|------|----------|
| **信任优先** | 品牌咨询是高客单价服务，用户需要先信任工具才愿意投入时间 | 采用商务感配色（深蓝/靛蓝为主色），避免过度娱乐化；报告页面结构严谨、数据可视化清晰 |
| **零学习成本** | 目标用户是非技术人群，40+老板可能对复杂界面有使用障碍 | 对话式交互为核心（参考RICH设计范式），像微信聊天一样自然；功能入口隐藏在对话流中，不强制用户理解产品结构 |
| **渐进式引导** | 用户不知道该问什么，需要AI主动提问引导 | 借鉴Notion AI的"图标+上下文提示"模式，AI主动抛出选择题而非开放式问题；每个步骤都有明确的下一步指引 |
| **过程透明** | AI生成诊断报告时，用户需要理解"为什么得出这个结论" | 借鉴Linear的"Why?"可展开链接和Claude的思考过程展示；报告中每个结论附带来源引用和推理链 |
| **结果可感知** | 用户需要看到"切实的产出"而非空泛建议 | 诊断报告以结构化卡片+可视化图表呈现；可导出PDF/分享链接，增强实体感 |

### 1.3 设计理念推导

基于蚂蚁AI设计小组提出的**RICH设计范式**（Role-Intention-Conversation-HybridUI），结合本产品特性：

- **Role（角色）**：AI扮演"品牌顾问"角色，语气专业但不居高临下，像一个经验丰富的朋友在给建议
- **Intention（意图）**：用户意图往往是模糊的（"我想做品牌"），需要通过AI互动逐步明确，而非要求用户一次性输入完整需求
- **Conversation（对话）**：核心交互载体是对话，但不是纯自由对话——是"结构化引导式对话"，AI用选择题、评分题等降低用户表达成本
- **HybridUI（混合界面）**：对话流中嵌入文件上传、图表展示、报告卡片等富组件，避免页面跳转，保持沉浸式体验

## 2. 对标品牌及原因

### 2.1 主要对标：Stripe（清爽商务）+ Notion（温和易用）的融合

| 对标品牌 | 借鉴维度 | 具体参考点 | 应用到本产品 |
|----------|----------|------------|--------------|
| **Stripe** | 配色体系、排版节奏、信任感营造 | 靛蓝主色(#533afd→调整为更稳重的深蓝)、浅色画布、8px间距网格、低圆角(4-8px) | 主色采用深靛蓝系，传递金融级专业可信感；报告页面参考Stripe的数据展示排版 |
| **Notion** | 交互引导、温和感、内容组织 | 图标+上下文提示引导用户操作、极简干净的界面、温暖灰调 | 对话页采用Notion式的轻量引导（灯泡图标+建议操作），降低用户使用焦虑 |
| **Linear** | 精准感、信息层级、过程透明 | "Why?"可展开解释、hairline边框分隔、紧凑信息密度 | 报告结论附带可展开的推理过程；整体界面采用细边框而非重阴影来区分层级 |
| **ChatGPT** | 对话交互基线 | 流式输出、自动滚动、输入框设计、历史记录侧边栏 | 对话核心交互参考ChatGPT，但优化为"引导式对话"而非完全自由对话 |
| **Claude** | Artifacts混合界面、思考过程展示 | 对话中嵌入可交互的内容卡片、展示推理步骤 | 诊断报告以Artifact卡片形式嵌入对话流，不跳转新页面 |

### 2.2 为什么选择这个融合方向

**目标用户画像决定了设计方向**：

```
中小企业老板 / 创业者
├── 年龄：30-55岁为主
├── 技术能力：低，习惯微信/钉钉，不习惯复杂SaaS
├── 决策心理：品牌咨询花费不菲，需要"专业可信"的第一印象
├── 使用场景：碎片化时间，可能手机使用
└── 核心诉求："别让我学怎么用，直接告诉我该做什么"
```

- **选Stripe而非Linear作为主色参考**：Linear的暗黑+荧光绿过于"极客"，对非技术用户有距离感；Stripe的浅色+靛蓝更符合商务人群对"专业"的认知
- **融入Notion的温和感**：纯Stripe风格偏冷峻，加入Notion的图标引导和温暖灰调，降低使用门槛
- **对话基线参考ChatGPT**：用户已有心智模型，降低学习成本

## 3. 配色基调

### 3.1 设计思路

采用**浅色模式为主**（Light Mode First），原因：
1. 目标用户（中小企业老板）日常使用微信、钉钉等均为浅色界面，浅色更符合习惯
2. 品牌咨询场景需要阅读大量文字报告，浅色背景更利于长时间阅读
3. 浅色界面更容易传递"专业、干净、可信"的感觉

### 3.2 主色板

#### 品牌主色 — 深靛蓝（Trust Indigo）

灵感来自Stripe的靛蓝(#533afd)，但调整为更深沉稳重的色调，传递"品牌咨询"的专业感而非"科技感"。

| 色彩角色 | 名称 | 色值 | 用途 |
|----------|------|------|------|
| 主色 | Trust Indigo | `#3B4BDF` | 主CTA按钮、品牌标识、链接、选中态、重点强调 |
| 主色-深 | Deep Indigo | `#2A36B8` | 主色Hover态、按下态 |
| 主色-浅 | Soft Indigo | `#7B85F0` | 次级强调、图标高亮 |
| 主色-背景 | Indigo Tint | `#EEF0FC` | 选中行背景、AI消息气泡背景、标签背景 |
| 主色-极浅 | Indigo Wash | `#F5F6FE` | 页面级大面积浅色背景区块 |

#### 辅助色 — 成长绿（Growth Green）

用于成功状态、正向反馈、进度指示，传递"品牌成长"的积极意象。

| 色彩角色 | 名称 | 色值 | 用途 |
|----------|------|------|------|
| 辅助色 | Growth Green | `#16A34A` | 成功状态、完成标记、正向评分 |
| 辅助色-浅 | Soft Green | `#DCFCE7` | 成功背景色、正向标签 |

#### 辅助色 — 警示橙（Alert Orange）

用于需要用户注意但不紧急的信息，如报告中的风险提示、待完善项。

| 色彩角色 | 名称 | 色值 | 用途 |
|----------|------|------|------|
| 辅助色 | Alert Orange | `#EA580C` | 风险提示、待办提醒、中等优先级标记 |
| 辅助色-浅 | Soft Orange | `#FFF7ED` | 警示背景色 |

#### 辅助色 — 危险红（Danger Red）

仅用于错误状态、删除操作、负面评分。

| 色彩角色 | 名称 | 色值 | 用途 |
|----------|------|------|------|
| 辅助色 | Danger Red | `#DC2626` | 错误状态、删除按钮、负面评分 |
| 辅助色-浅 | Soft Red | `#FEF2F2` | 错误背景色 |

### 3.3 中性色板

参考Stripe的冷灰系中性色，但略微加入暖调，避免过于冰冷。

| 色彩角色 | 名称 | 色值 | 用途 |
|----------|------|------|------|
| 标题文字 | Ink | `#0F172A` | 一级标题、重要数据、页面主标题 |
| 正文文字 | Slate | `#334155` | 正文段落、列表内容、表格数据 |
| 次要文字 | Mute | `#64748B` | 次要说明、时间戳、辅助标签 |
| 占位文字 | Fog | `#94A3B8` | 输入框placeholder、禁用态文字 |
| 极浅文字 | Mist | `#CBD5E1` | 极次要信息、分割线文字 |
| 边框-主 | Border | `#E2E8F0` | 卡片边框、输入框边框、分割线 |
| 边框-浅 | Hairline | `#F1F5F9` | 表格行分割、卡片内部分割 |
| 背景-卡片 | Surface | `#FFFFFF` | 卡片背景、对话气泡背景（用户侧） |
| 背景-页面 | Canvas | `#FAFBFC` | 页面底色，微偏冷的白 |
| 背景-次 | Canvas Soft | `#F8FAFC` | 次级背景区域、侧边栏背景 |
| 背景-悬浮 | Float | `#F1F5F9` | Hover态背景、选中态浅背景 |

### 3.4 渐变色

用于品牌标识、报告封面、空状态插画等品牌表达场景，不用于功能性UI。

| 名称 | 色值 | 用途 |
|------|------|------|
| Brand Gradient | `linear-gradient(135deg, #3B4BDF 0%, #7B85F0 100%)` | Logo背景、报告封面、品牌头图 |
| Trust Gradient | `linear-gradient(180deg, #EEF0FC 0%, #FFFFFF 100%)` | 报告页顶部渐变背景 |

### 3.5 色彩使用规则

1. **一屏一主色**：每个视图只有一个主色CTA，参考Stripe的"single voltage CTA"原则
2. **主色克制使用**：主色(Trust Indigo)仅用于需要用户行动的元素（按钮、链接、选中态），不做装饰用途
3. **文字不用彩色**：正文始终使用中性色板，彩色仅用于状态标签和图标
4. **对比度达标**：所有文字与背景对比度满足WCAG AA标准（4.5:1以上）

## 4. 字体方案

### 4.1 设计思路

- **中文用户为主**：中文字体选择优先，英文/数字字体作为搭配
- **屏幕阅读优化**：选择为屏幕显示优化的字体，非印刷字体
- **专业但不冰冷**：避免衬线体（过于传统），选择现代无衬线体但字重适中

### 4.2 字体方案

#### 中文字体：思源黑体（Source Han Sans / Noto Sans CJK SC）

| 字重 | 用途 | 说明 |
|------|------|------|
| Regular (400) | 正文 | 段落、列表、表格、对话内容 |
| Medium (500) | 次级标题 | 卡片标题、小节标题、按钮文字 |
| Semibold (600) | 标题 | 页面标题、报告大标题 |
| Bold (700) | 强调 | 数据数字、关键结论（谨慎使用） |

**选择理由**：
- 开源免费，无授权风险
- Adobe + Google联合开发，字形规范统一
- 屏幕显示清晰，笔画粗细适中，比苹方稍显稳重
- 中英文混排时与Inter高度协调

#### 英文/数字字体：Inter

| 字重 | 用途 | 说明 |
|------|------|------|
| Regular (400) | 英文正文 | 与中文Regular搭配 |
| Medium (500) | 英文次级标题 | 与中文Medium搭配 |
| Semibold (600) | 英文标题 | 与中文Semibold搭配 |
| Bold (700) | 数字强调 | 数据展示、评分数字 |

**选择理由**：
- Linear、Stripe等顶级SaaS产品均在使用的字体
- 为屏幕显示优化，字距和x-height经过精心调整
- Variable Font支持，加载性能好
- 与思源黑体混排视觉协调

#### 等宽字体：JetBrains Mono

仅用于技术性内容（如知识库文件名、代码片段、API密钥等极少场景）。

### 4.3 字号体系

基于Stripe的8px网格系统，采用1.25比例（Major Third）的字号阶梯：

| Token | 字号 | 行高 | 字重 | 用途 |
|-------|------|------|------|------|
| `text-display` | 36px | 44px | 600 | 报告封面大标题 |
| `text-h1` | 28px | 36px | 600 | 页面主标题 |
| `text-h2` | 22px | 30px | 600 | 区块标题、报告章节标题 |
| `text-h3` | 18px | 26px | 500 | 卡片标题、小节标题 |
| `text-body-lg` | 16px | 26px | 400 | 对话内容、报告正文（大） |
| `text-body` | 15px | 24px | 400 | 默认正文、列表项 |
| `text-body-sm` | 14px | 22px | 400 | 次要正文、表格内容 |
| `text-caption` | 13px | 20px | 400 | 标签、时间戳、辅助说明 |
| `text-micro` | 12px | 18px | 500 | 徽章、状态标记、按钮内文字 |

### 4.4 字距规则

参考Linear的字距调整策略：
- 大标题（28px+）：`letter-spacing: -0.02em`（紧凑，提升标题感）
- 中标题（18-22px）：`letter-spacing: -0.01em`
- 正文（14-16px）：`letter-spacing: 0`（默认）
- 微小文字（12-13px）：`letter-spacing: 0.01em`（略微放松，提升可读性）

### 5.1 主图标库：Lucide Icons

| 项目 | 说明 |
|------|------|
| 名称 | Lucide Icons |
| 网址 | https://lucide.dev |
| 风格 | 线性、简洁、圆角端点 |

- 知识库图标

| 规范项 | 标准 |
|--------|------|

## 6. 整体风格描述

### 6.1 风格关键词

> **专业 · 温暖 · 可信赖 · 清爽**

### 6.2 风格描述

**专业（Professional）**
- 深靛蓝主色传递商务专业感，如 Stripe 般的金融级信任
- 报告页面结构严谨，数据可视化清晰，参考 Stripe 的表格对齐和数字排版
- 字体字重适中，不用过轻（300）或过重（700+），保持稳重

**温暖（Warm）**
- 中性色略带暖调（Canvas #FAFBFC 而非纯冷灰），避免冰冷感
- AI对话语气友好但不轻浮，像一位经验丰富的品牌顾问在聊天
- 借鉴 Notion 的图标引导（灯泡、笔等），用视觉元素传递"我在帮你"的感觉
- 空状态、引导提示使用柔和插画或图标，不用冷硬的系统提示

**可信赖（Trustworthy）**
- 每个AI结论附带"Why?"可展开解释，参考 Linear 的过程透明设计
- 诊断报告有明确的数据来源和评分依据
- 报告格式正式、可导出PDF，增强实体感和专业感
- 人工审核环节明确标识，让用户知道"有真人在把关"

**清爽（Clean）**
- 大量留白，信息密度适中偏低（比 Linear 宽松，适合非技术用户）
- 细边框（1px Hairline）分隔层级，几乎不用阴影
- 一屏一主色，视觉焦点明确
- 圆角适中（6-12px），不过圆（显幼稚）也不过方（显冰冷）

### 6.3 风格不做什么

- 不做暗色模式（MVP阶段）：目标用户习惯浅色，暗色模式增加开发成本且可能传递"技术感"而非"商务感"
- 不做大型插画/吉祥物：品牌咨询是严肃决策，过度装饰降低专业感
- 不用渐变色做功能UI：渐变仅限品牌标识和报告封面
- 不用动画炫技：动效仅用于状态反馈（loading、成功提示等），不做装饰性动画
- 不用emoji作为UI元素：非技术用户可能感知不够专业

## 9. 响应式设计策略

### 9.1 断点定义

参考主流SaaS产品的断点体系：

| 断点名称 | 宽度 | 设备 | 说明 |
|----------|------|------|------|
| `sm` | < 640px | 手机竖屏 | 单列布局，底部Tab栏 |
| `md` | 640px - 1024px | 平板 / 手机横屏 | 单列布局，可展开侧边栏 |
| `lg` | 1024px - 1280px | 小屏笔记本 | 双列布局（侧边栏+主内容） |
| `xl` | > 1280px | 桌面显示器 | 双列布局，主内容区最大宽度限制 |

### 9.2 各断点布局策略

#### 桌面端（lg / xl）— 默认体验

```
┌────────┬───────────────────────────┐
│        │                           │
│ 侧边栏  │      主内容区              │
│ 240px  │      max-width: 768px     │
│ 固定    │      居中                   │
│        │                           │
└────────┴───────────────────────────┘
```

- 侧边栏固定展开，240px宽度
- 主内容区居中，最大宽度768px（对话内容最佳阅读宽度）
- 顶部导航栏64px高度

#### 平板端（md）

```
┌───────────────────────────────────┐
│ [☰]  顶部导航栏                     │
├───────────────────────────────────┤
│                                   │
│         主内容区                    │
│      max-width: 640px             │
│         居中                       │
│                                   │
└───────────────────────────────────┘
```

- 侧边栏默认收起，通过左上角汉堡菜单展开（抽屉式）
- 主内容区最大宽度640px

#### 移动端（sm）

```
┌─────────────────┐
│ [☰]  品牌智囊  [👤]│
├─────────────────┤
│                 │
│   主内容区      │
│   全屏宽度      │
│   padding: 16px │
│                 │
├─────────────────┤
│ [💬] [📊] [📚] [⚙️]│
└─────────────────┘
```

- 顶部导航栏精简：汉堡菜单 + Logo + 头像
- 底部Tab栏：对话、报告、知识库、设置
- 对话内容全屏宽度，左右padding 16px
- 侧边栏改为全屏抽屉
- 输入框固定在底部Tab栏上方

### 9.3 移动端特殊处理

| 场景 | 桌面端 | 移动端调整 |
|------|--------|------------|
| 对话消息 | 最大宽度70% | 最大宽度85%，减少留白 |
| 选择题选项 | 横向2列排列 | 纵向单列排列 |
| 报告评分卡 | 多维并列展示 | 纵向堆叠，可滑动查看 |
| 文件上传 | 拖拽区域较大 | 拖拽区域缩小，主要靠点击 |
| 报告导出 | 顶部操作栏 | 底部固定操作栏 |
| 知识库列表 | 表格/列表布局 | 卡片纵向列表 |

## 10. 交互设计规范

### 10.1 对话交互规范

#### 10.1.1 消息发送

| 场景 | 交互 | 说明 |
|------|------|------|
| 文字输入 | 输入框 → 回车/点击发送按钮 | Shift+回车换行 |
| 选择题 | 点击选项 → 自动发送 | 选中后高亮0.3s后自动发送 |
| 文件上传 | 点击📎/拖拽文件 → 自动上传 | 上传完成后AI自动确认 |
| 快捷操作 | 点击快捷卡片 → 自动填入输入框并发送 | 等同于发送预设问题 |

#### 10.1.2 AI回复展示

| 阶段 | 展示方式 | 说明 |
|------|----------|------|
| 思考中 | 三点脉冲动画 + "正在思考..." | 超过3秒显示具体步骤 |
| 生成中 | 流式逐字输出 | 参考ChatGPT，文字渐显效果 |
| 生成完成 | 轻微震动反馈 | 移动端Haptic反馈 |
| 内容较长 | 自动滚动到底部 | 用户上滑时暂停，显示"↓新内容" |
| 包含图表 | 嵌入式图表卡片 | 在对话流内展示，不跳转 |

#### 10.1.3 消息操作

- **复制**：每条AI消息底部提供复制按钮
- **重新生成**：AI消息底部提供"重新生成"按钮
- **反馈**：AI消息底部提供👍/👎反馈按钮
- **引用追问**：长按/选中AI消息部分文字，弹出"针对这段追问"操作

### 10.2 状态反馈规范

| 状态 | 视觉表现 | 示例 |
|------|----------|------|
| 加载中 | 骨架屏 + 脉冲动画 | 报告生成中、知识库学习中 |
| 成功 | 绿色Toast提示 + ✅图标 | "文件上传成功"、"报告已下载" |
| 失败 | 红色Toast提示 + ❌图标 + 重试按钮 | "上传失败，请重试" |
| 警告 | 橙色Toast提示 + ⚠️图标 | "文件格式不支持" |
| 进行中 | 进度条 + 百分比 + 状态文字 | "正在学习知识库... 56%" |

### 10.3 按钮规范

| 类型 | 样式 | 用途 |
|------|------|------|
| Primary | 主色背景 + 白字 + 6px圆角 | 页面主操作（发送、提交、下载） |
| Secondary | 白色背景 + 主色边框 + 主色字 | 次级操作（预览、取消） |
| Ghost | 透明背景 + 次要文字色 | 第三级操作（更多、关闭） |
| Danger | 红色背景 + 白字 | 删除、危险操作 |
| Icon | 透明背景 + 图标 | 工具栏操作（复制、分享） |

### 10.4 表单规范

| 元素 | 样式 | 说明 |
|------|------|------|
| 输入框 | 白底 + 1px边框 + 6px圆角 | Focus态：主色边框 + 阴影环 |
| 下拉框 | 同输入框 + 右侧箭头图标 | 展开时显示在Popover中 |
| 复选框 | 16px + 4px圆角 | 选中态：主色背景 + 白✅ |
| 单选框 | 16px圆形 | 选中态：主色边框 + 主色中心点 |
| 开关 | 44×24px | 开启态：主色背景 |

### 10.5 动效规范

| 场景 | 动效 | 时长 |
|------|------|------|
| 按钮Hover | 背景色渐变 | 150ms |
| 卡片展开 | 高度+透明度过渡 | 250ms ease-out |
| 页面切换 | 淡入淡出 | 200ms |
| 消息出现 | 从下方滑入+淡入 | 300ms ease-out |
| Loading | 脉冲动画 | 1.2s循环 |
| 进度条 | 宽度过渡 | 400ms ease-out |
| Toast | 从顶部滑入+淡入 | 250ms ease-out |

## 11. 竞品调研附录

### 11.1 ChatGPT 对话界面分析

| 维度 | 观察要点 | 对本产品的启发 |
|------|----------|----------------|
| 布局 | 左侧历史对话栏 + 右侧对话区，对话区居中窄列 | 采用相同布局，用户已有心智模型 |
| 输入框 | 固定底部，附带工具按钮（附件、语音等） | 输入框固定底部，附带文件上传按钮 |
| 消息气泡 | 无传统气泡，用头像+内容块区分 | 采用气泡设计增强对话感（更适合非技术用户） |
| 流式输出 | 逐字显示 + 渐变透明度动画 | 采用相同流式输出方案 |
| 自动滚动 | 用户上滑时暂停，显示"↓新内容"按钮 | 完全复用此交互 |
| 历史记录 | 按时间分组，AI自动生成标题 | 采用相同分组+自动标题方案 |

### 11.2 Notion AI 界面分析

| 维度 | 观察要点 | 对本产品的启发 |
|------|----------|----------------|
| 引导方式 | 灯泡/笔等图标 + 上下文提示引导用户操作 | 快捷入口使用图标+文字卡片 |
| 嵌入式AI | AI功能嵌入工作流，不独立打开 | 报告卡片嵌入对话流，不跳转 |
| 极简设计 | 干净、留白充足、视觉噪音少 | 对话页保持极简，减少视觉干扰 |
| 活动记录 | AI变更可追溯、可撤销 | 报告版本可追溯 |

### 11.3 Linear 设计系统分析

| 维度 | 观察要点 | 对本产品的启发 |
|------|----------|----------------|
| 配色 | 暗色为主，极简，单一主色强调 | 改为浅色（更适合目标用户），但保留"单一主色"原则 |
| 字体 | Inter Variable, 字重上限590, 紧凑字距 | 采用Inter + 思源黑体，字重不超过600 |
| 边框 | 0.5-1px hairline边框替代阴影 | 使用1px细边框分隔卡片层级 |
| 圆角 | 三级圆角体系(6px/12px/pill) | 采用类似的有限圆角体系(4/6/8/12/16px) |
| "Why?"链接 | AI建议附带可展开解释 | 报告结论附带可展开推理过程 |
| 信息密度 | 紧凑但不拥挤 | 略微放宽（非技术用户需要更多呼吸感） |

### 11.4 Stripe 设计系统分析

| 维度 | 观察要点 | 对本产品的启发 |
|------|----------|----------------|
| 主色 | 靛蓝 #533afd，单一品牌色驱动所有CTA | 主色调整为更深稳的 #3B4BDF |
| 字体 | Sohne (weight 300-400), 负字距 | 思源黑体 + Inter替代, 字重400-600 |
| 间距 | 8px网格, 2-96px范围 | 完全采用8px网格间距体系 |
| 圆角 | 0-16px, 低圆角为主 | 采用4-16px圆角体系 |
| 表格数据 | tnum等宽数字, 精确对齐 | 报告数据采用等宽数字对齐 |
| 阴影 | 极克制, 以边框为主 | 阴影仅用于悬浮元素和弹窗 |
| 渐变 | 仅用于品牌营销页面 | 渐变仅用于报告封面和Logo |

### 11.5 Kimi / 通义千问 界面分析

| 维度 | 观察要点 | 对本产品的启发 |
|------|----------|----------------|
| 长文本处理 | Kimi擅长长文本，支持超长文件上传 | 知识库支持大文件上传和学习 |
| 多模态 | 通义千问支持图片、文档等多种输入 | 文件上传支持多格式（PDF/Word/PPT/图片/Excel） |
| PPT生成 | 通义千问有AI PPT功能 | 报告可考虑生成可编辑的PPT版本（未来迭代） |
| 中文优化 | 国产AI在中文理解和表达上更自然 | AI对话语气针对中文用户优化，避免翻译腔 |

### 11.6 AI UX 设计模式总结

基于Mantlr的AI Agent UX模式研究和蚂蚁RICH设计范式，提炼出本产品采用的核心模式：

| 模式 | 来源 | 应用场景 |
|------|------|----------|
| Goal-First Onboarding | Cursor | 首次进入时以对话方式了解用户品牌情况，不填表单 |
| Explainability on Demand | Linear | 报告结论附"Why?"可展开链接 |
| Action Preview | Claude Code | 生成报告前先展示大纲预览，用户确认后再生成 |
| Activity Feed | Notion AI | 记录AI学习和生成全过程，可追溯 |
| Graceful Error Recovery | Cursor | AI出错时明确说明问题并提供修正方案 |
| Contextual Guardrails | Gmail | 生成报告=低风险自动执行；导出/分享=需确认 |
| Multimodal Handoff | Claude Artifacts | 对话中嵌入报告卡片，对话与可视化无缝切换 |
